# Enterprise Multi-Site Network Infrastructure Deployment

## Overview
Demonstrated hands-on experience designing and deploying a simulated enterprise multi-site campus network using Cisco Packet Tracer. This project involved designing a hierarchical network topology connecting a main campus, branch campus, and cloud environment while implementing VLAN segmentation, Layer 2 switching, Layer 3 switching, inter-VLAN routing, dynamic routing, DHCP services, and centralized server connectivity.

---

## Environment / Tech Stack

- Cisco Packet Tracer
- Cisco 2911 Integrated Services Routers
- Cisco Catalyst 2960 Switches
- Cisco Catalyst 3650-24PS Layer 3 Switches
- Server-PT
- PC-PT
- Printer-PT

---

## Enterprise Network Deployment

- Designed a multi-site enterprise campus network topology consisting of a main campus, branch campus, and cloud network
- Deployed routers, Layer 3 switches, Layer 2 access switches, servers, workstations, and network printers
- Established LAN and WAN connectivity using Ethernet and Serial DCE connections
- Segmented departments into dedicated VLANs across both campus locations
- Configured Layer 2 access ports for departmental VLAN membership
- Configured Layer 3 switch interfaces to connect access switches
- Configured IEEE 802.1Q trunk links between routers and Layer 3 switches
- Assigned IPv4 addressing to WAN interfaces
- Configured Router-on-a-Stick for inter-VLAN routing
- Implemented DHCP services for automatic IP address assignment across all VLANs
- Configured static IPv4 addressing for enterprise servers
- Implemented RIPv2 dynamic routing between the main campus, branch campus, and cloud router
- Configured client devices to obtain network configuration automatically through DHCP
- Validated end-to-end connectivity between VLANs, campuses, and enterprise servers

## Key Takeaways

Enterprise campus networks require structured network segmentation, scalable routing, centralized address management, and reliable WAN connectivity. This project demonstrates how VLANs, Layer 3 switching, inter-VLAN routing, RIPv2, DHCP, and enterprise server integration work together to provide secure, scalable, and manageable communication across multiple campus locations.
